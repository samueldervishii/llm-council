Release notes

What's new in Étude, from oldest to newest.

Étude is built in the open. Every change shipped to production is summarized below. For the full diff of any release, follow the GitHub link at the bottom of each entry.

v1.0.0

April 27, 2026 Latest Stable

First stable release. The product graduates from Cortex (preview) into Étude — new name, new mark, and a unified typographic system. Underneath, this is the v0.2 polish hardened: a real dark mode, a stable PWA install, mobile UX fixes across the surface, the new Chats page, and a tightened auth flow.

Rebrand & visual identity

  • New name — Cortex is now Étude. The new mark replaces the previous logo across the web app, the marketing site, and the PWA install surfaces.
  • One typographic system — every surface (app and site) now sets in Fraunces. The previous mix of Inter, JetBrains Mono, and ad-hoc system stacks is gone; the Google Fonts request was trimmed accordingly so first paint is a touch lighter.
  • A few identifiers — the GitHub repo, the Vercel subdomains, the support email, and saved-session storage keys — keep their cortex name on purpose. Renaming any of those would either break working systems or sign people out.

Highlights

  • Stable PWA install on iOS and Android, with proper status-bar handling on iOS so the top bar and sidebar no longer collide with the notch.
  • Full dark mode — rebuilt palette, fixed sidebar and welcome surfaces, and patched the user bubble so it reads cleanly on both themes.
  • Mobile UX pass across settings, the top bar, the welcome screen, and the sidebar hamburger nav.
  • New Chats page with bulk select and a confirm-before-delete flow, reachable from the sidebar.
  • Auth hardening: fixed a cold-start path that briefly let unauthenticated users land on protected screens.
  • Refreshed website aligned with the in-app terracotta paper palette, plus updated copy.

Maintenance

  • Bumped python-dotenv from 1.2.1 to 1.2.2.

v0.2

April 2026 Preview

The preview release. Most of v1.0's polish landed here first — this is where the mobile and theme work was originally introduced and where the auth fix was shipped.

What's changed

  • Mobile nav hamburger now actually opens and closes the sidebar.
  • Mobile UX fixes for settings, top bar, and welcome screen.
  • Landing site aligned with the frontend's paper + terracotta palette.
  • Refreshed website copy.
  • Reserved space for the iOS status bar in the top bar and sidebar.
  • Swapped logo across the app and website.
  • Dark mode for the PWA, including a fix for the iOS status bar on /status.
  • Rebuilt the dark-mode palette and patched sidebar/welcome surfaces.
  • Removed the white halo behind the mobile welcome input and cleaned the user bubble.
  • Fixed the dark-mode user bubble and a cold-start auth bypass.
  • Bumped python-dotenv from 1.2.1 to 1.2.2.
